# Anamnesis UI Revamp — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Gradio WebUI with a modern forensic-intelligence dashboard (Lite⇄Pro), fed by a thin FastAPI layer over the *unchanged* forensic engine, served alongside the agent on Alibaba ECS.

**Architecture:** A new `api/` (FastAPI) wraps the frozen engine — `POST /api/assess` returns a card-shaped verdict JSON, `POST /api/chat` streams the agent over SSE, plus `/api/graph` and `/api/price`. A new `frontend/` (Vite SPA) renders the mockup-v4 dashboard against that API. nginx on the ECS serves the static build + proxies `/api`.

**Tech Stack:** Backend — FastAPI + uvicorn + sse-starlette (over existing qwen-agent/mcp/pymongo engine). Frontend — Vite + React + TypeScript + Tailwind + shadcn/ui + Framer Motion + Vitest/Testing Library.

## Global Constraints

- **Engine frozen:** do NOT modify `src/anamnesis/**` or `mcp/**` logic. The API composes card JSON from existing engine outputs; new serialization lives in `api/`.
- **Deps pinned:** `qwen-agent==0.0.34`, `mcp==1.12.4` — do not unpin (unpinned mcp backtracks → slow/timeout build).
- **Secrets:** never hardcode; `ANAMNESIS_DASHSCOPE_API_KEY`, `ANAMNESIS_HELIUS_API_KEY`, `ANAMNESIS_MONGODB_URI` from env only. Helius key must never appear in logs/responses (engine already scrubs at `_rpc`).
- **Backend on Alibaba ECS** (hard hackathon requirement). Everything on one box; no cross-origin.
- **Verdict is the hero; autopilot is a supporting beat** (MemoryAgent framing — see `~/Documents/secret/strategy/anamnesis/TRACK-DECISION.md`).
- **Visual source of truth:** mockup v4 (`scratchpad/anamnesis-dashboard-mockup.html`) — exact markup, tokens, animation, tooltips, Lite/Pro. Frontend tasks port from it; do not re-invent styling.
- Python: `.venv/bin/python -m pytest -q` must stay green (currently 236). Ruff clean: `.venv/bin/ruff check src tests app.py mcp api`.

```python
# api/cards.py
"""Pure serializers: engine verdict/result -> card-shaped JSON for the dashboard.
No network, no engine mutation — composes over assess_and_act's dict output."""
from __future__ import annotations


def _rugs(remembered: list[dict]) -> list[dict]:
    return [{"mint": e["dst"], "date": e.get("valid_from")}
            for e in remembered if e.get("type") == "RUGGED" and e.get("method") == "first_party"]


def verdict_card(result: dict, deployer: str | None) -> dict:
    score = float(result.get("score", 0.0))
    return {
        "level": result.get("level"),
        "score": score,
        "mint": result.get("mint"),
        "deployer": deployer,
        "rationale": result.get("rationale"),
        "provenance": {
            "first_party": round(score, 2) if result.get("level") in ("HIGH", "MEDIUM") else None,
            "derived": None,   # capped-MEDIUM tier, context in UI only
            "claimed": None,   # context only, never scores
        },
        "memory_rugs": _rugs(result.get("remembered", [])),
        "signals": result.get("cited_signals", []),
        "acted": result.get("acted", False),
        "watchlisted": result.get("watchlisted"),
        "alert": result.get("alert"),
    }
```
